> AI 时代，会议正在从“沟通工具”变成“上下文生产工具”。它会让公司更容易记住那些过去会蒸发的讨论、分歧和判断过程，但如果所有话都被记录、检索和长期保存，人也可能变得更谨慎、更沉默。

## 公司资产正在从文档迁移到会议

最近看到一个观点，越想越觉得有后劲：

> AI 时代，公司的核心资产正在从“写下来的知识”，迁移到“被捕捉、理解和持续复用的上下文”。

以前我们说公司资产，通常想到的是文档、Wiki、工单。它们很重要，因为它们把结果写下来了，把流程固化下来了，也让后来的人可以查。但很多真正有价值的信息，可能根本没写进文档里。比如：

用户在访谈里随口说的一句话/产品评审里某个人提出的反对意见

这些东西过去经常开完会就散了。

文档里留下的是结论，但会议里发生的是过程。结论当然重要，但过程里有更多真实的上下文。

David Haber 在[*《Everything Is Recorded Now》*](https://x.com/dhaber/status/2064711613714735141)里也提到，AI 正在让大量工作讨论默认被记录，会议、客户电话、产品评审、领导层讨论这些口头交流，正在变成新的、可搜索和可查询的组织记录。

## 会议会变成一种新的 Prompt

这里说的 prompt，不是简单给 AI 写一句“帮我总结一下”。更准确地说，prompt 是在给 AI 提供背景、目标、约束、偏好、判断标准和上下文。

而一场高质量会议，天然就包含这些东西。

| 会议里的内容 | 对 Agent 的价值 |
| --- | --- |
| 具体例子 | 帮助理解真实场景，而不是只看抽象结论 |
| 用户原话 | 保留一手反馈，减少二次转述的信息损耗 |
| 分歧和反对意见 | 理解方案边界和风险来源 |
| 临时判断 | 捕捉文档里不会写出来的经验和直觉 |
| 决策过程 | 知道为什么选 A，而不是只知道最后选了 A |

以前这些信息对人有用，但很难沉淀。现在 AI 可以把它们记录下来、结构化、关联到已有上下文里。

这件事的想象空间其实很大。

- 需求沟通结束后，agent 自动提炼洞察和 PRD 初稿。
- 项目讨论结束后，agent 自动更新项目状态和风险提醒。
- 每次会议结束后，agent 都更懂你一点，更懂团队一点。

所以会议的结束，可能不再是工作的结束，而是 agent 开始工作的地方。

## 但人会不会越来越沉默？

如果所有会议都默认被录制、转写、总结、检索、沉淀到组织 memory 里，人会不会慢慢变得不像在讨论，而像在正式发言？这就是我对“Everything is recorded now”的复杂感受：AI 会让组织更聪明，但如果没有边界和安全感，也可能让组织更沉默。

这里最讽刺的地方是：如果大家因为被记录而不敢说，agent 得到的上下文也会变差。

我们可能记录了更多会议，却失去了更多真实信息。

## 不是录不录，而是怎么保护表达空间

更合理的方式，可能是给会议分层？

| 会议类型 | 适合的记录方式 | 原因 |
| --- | --- | --- |
| 普通同步会 | 默认记录结论和行动项 | 减少重复沟通，方便追踪执行 |
| 业务会议 | 尽量保留逐字稿 | 需求方原话和真实反馈很有价值 |
| 产品评审 | 记录过程、分歧和最终取舍 | 帮助 agent 理解为什么这样决策 |
| Brainstorm | 可以只沉淀方向，不永久保存全部发言 | 保护半成品想法和自由表达 |
| 人事 / 绩效 / 冲突沟通 | 默认敏感，只记录必要结论 | 需要更严格的权限和边界 |

未来真正重要的，可能不是一个“录音开关”，而是一套上下文权限管理。不是所有东西都应该被永久记住，不要让“记录一切”变成“审视一切”。最好的状态，可能不是把所有话都保存下来，而是知道什么值得被记住，什么应该被允许只是一次现场的、临时的、不成熟的表达。

> 如果会议真的成为 AI 时代的重要 prompt，那我们要设计的不只是更好的会议工具，还有一个让人愿意继续说真话的环境。

这可能才是 AI 时代开会最难，也最值得认真想的地方。
